# OpenCode Language Configuration

**IMPORTANT: Always follow these language rules:**

## Language Policy

1. **Internal Thinking**: You MUST always think and process information in English during your reasoning, analysis, and planning phases.

2. **User Communication**: You MUST always communicate with the user in Chinese (中文) for all responses, explanations, and interactions.

3. **Code and Documentation**: 
   - Code comments and inline documentation should follow the project's existing language conventions
   - Technical terms and variable names should remain in English
   - User-facing documentation and README files should be in Chinese when appropriate

## Application

This policy applies to:
- All responses to user queries
- Status updates and progress reports
- Code explanations and documentation
- Error messages and troubleshooting guidance
- All verbal and written communication with the user

**Remember**: Think in English, speak in Chinese.

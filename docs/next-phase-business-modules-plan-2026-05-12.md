# 下一阶段业务模块建设计划

日期：2026-05-12

## 背景

当前系统已经具备客户、渠道、线索、商机、项目、合同、销售目标、工单派工、报表和权限治理等核心能力。下一阶段不做泛化扩张，重点补齐销售日常运营和业务协同能力。

本阶段目标模块：

- 报价/价格/方案管理
- 任务/日程/提醒中心
- 通知中心
- 附件与文档管理
- 数据导入导出
- 客户联系人与组织关系深化
- 日报/周报管理
- 角色化工作台

## 建设原则

- 以销售、业务管理者、渠道运营、财务、技术员、管理员的实际工作流为中心。
- 优先补齐业务闭环中缺失的操作入口，而不是只做静态数据表。
- 新模块必须接入认证、权限策略、操作日志、分页上限、错误处理和测试。
- 前端页面必须具备加载态、空状态、错误提示和权限控制。
- 能复用现有客户、渠道、线索、商机、项目、合同、销售任务、工单数据时，不重复建孤立数据。
- 团队范围优先使用新增 `department_manager_id` 部门负责人关系，`sales_leader_id` 仅作为历史兼容字段。

## 模块 1：报价/价格/方案管理

### 目标

补齐商机到合同之间的报价过程，支持销售人员形成可追踪、可审批、可转合同的报价记录。

### 核心能力

- 产品价格表
- 报价单创建、编辑、复制、作废
- 报价明细：产品、数量、单价、折扣、税率、合计
- 报价版本管理
- 报价关联客户、商机、项目
- 报价转合同
- 折扣超限标记，为后续审批流预留入口
- 报价 PDF/Excel 导出预留

### 验收口径

- 商机详情可查看关联报价。
- 报价单金额自动汇总，明细变更后总额同步。
- 报价单可转合同，并保留来源关系。
- 折扣、金额、状态流转有后端规则校验。

## 模块 2：任务/日程/提醒中心

### 目标

将跟进记录、销售任务、合同到期、回款提醒、工单 SLA 等事项沉淀为统一待办和日程能力。

### 核心能力

- 个人待办列表
- 日程视图：今日、本周、本月
- 跟进提醒
- 合同到期提醒
- 回款逾期提醒
- 工单 SLA 提醒
- 待办完成、延期、关闭
- 与客户、商机、项目、合同、工单关联

### 验收口径

- 不同角色登录后只看到自己有权限处理的待办。
- 业务对象产生的关键日期能进入提醒中心。
- 完成或关闭待办后，工作台和提醒中心同步更新。

## 模块 3：通知中心

### 目标

建立统一通知入口，承接系统事件、业务提醒和外部集成消息。

### 核心能力

- 站内通知列表
- 未读/已读状态
- 批量已读
- 通知跳转到关联业务对象
- 通知类型：系统、业务、告警、工单、销售任务、合同、回款
- 飞书通知扩展预留
- 通知订阅规则预留

### 验收口径

- 通知能按用户维度查询。
- 通知跳转路径准确，不出现无权限对象泄露。
- 通知创建失败不影响主业务事务。

## 模块 4：附件与文档管理

### 目标

为客户、商机、项目、合同、报价、工单等对象提供统一附件能力。

### 核心能力

- 文件上传、下载、删除
- 文件类型、大小限制
- 附件关联业务对象
- 附件分类：合同、报价、客户资质、项目资料、工单照片、验收单、发票附件
- 上传人、上传时间、文件大小、文件名记录
- 权限校验：只能访问授权业务对象的附件

### 验收口径

- 至少支持客户、合同、工单三个对象的附件挂载。
- 无权限用户不能下载或删除附件。
- 删除业务对象时附件关系处理明确，不产生孤儿引用。

## 模块 5：数据导入导出

### 目标

提升运营效率，支持批量导入和标准化导出。

### 核心能力

- 模板下载
- 客户批量导入
- 线索批量导入
- 销售目标批量导入
- 合同/回款相关导出预留
- 导入预校验
- 导入错误报告
- 导入任务记录

### 验收口径

- 导入失败时能返回到行级错误。
- 重复数据处理规则明确。
- 导入任务可追踪，不直接吞掉异常。

## 模块 6：客户联系人与组织关系深化

### 目标

强化客户侧关系经营能力，支持销售理解客户组织结构和关键决策链。

### 核心能力

- 客户多联系人
- 主联系人标记
- 联系人岗位、部门、角色、联系方式
- 决策链/影响力标记
- 联系人跟进历史
- 客户集团、分公司、子公司关系
- 联系人与商机、项目、合同关联预留

### 验收口径

- 客户详情页可维护多个联系人。
- 一个客户最多一个主联系人。
- 联系人信息可被线索、商机、项目、合同选择引用。

## 模块 7：日报/周报管理

### 目标

沉淀销售、渠道运营、技术服务等角色的周期性工作复盘，支持管理者查看团队工作状态。

详细设计见 [日报/周报与角色化工作台设计](daily-weekly-report-and-role-dashboard-design-2026-05-13.md)。

### 核心能力

- 日报创建、编辑、提交、撤回
- 周报创建、编辑、提交、撤回
- 日报自动结构化汇总当天系统操作
- 周报自动聚合本周日报
- 备注字段支持灵活补充系统外事项、风险和计划
- 管理者查看下属日报/周报
- 评论或反馈预留

### 验收口径

- 销售人员可基于当日系统操作生成结构化日报草稿。
- 周报可从本周日报自动聚合生成。
- 管理者可按人员、部门、日期查看日报/周报。
- 日报/周报权限与组织或角色范围一致。
- 团队范围基于部门负责人关系判断。
- 日报/周报暂不强制提交。

## 模块 8：角色化工作台

### 目标

当前工作台主要面向销售角色。下一阶段应按角色提供不同首页视图，让用户登录后直接看到最相关的待办、指标和风险。

详细设计见 [日报/周报与角色化工作台设计](daily-weekly-report-and-role-dashboard-design-2026-05-13.md)。

### 角色视图

| 角色 | 工作台重点 |
|------|------------|
| 管理员 | 系统健康、用户、权限、告警、操作日志 |
| 业务管理者 | 销售目标完成、团队业绩、销售漏斗、重点商机、逾期事项 |
| 销售人员 | 我的客户、我的线索/商机、销售任务、待跟进、日报周报 |
| 财务人员 | 合同、回款进度、逾期提醒、客户财务视图 |
| 技术员 | 我的工单、派工状态、SLA、装机和服务评价 |
| 渠道运营 | 独立角色 `channel_ops`，展示渠道绩效、渠道跟进、渠道培训、渠道关联客户、日报周报 |

### 验收口径

- 登录后根据角色展示不同工作台模块。
- 无权限模块不展示，后端接口也不能越权返回数据。
- 工作台卡片能跳转到对应业务页面。

## 推荐实施批次

| 批次 | 模块 | 理由 |
|------|------|------|
| P1 | 角色化工作台、任务/日程/提醒中心、通知中心 | 提升所有角色日常使用效率，并为后续模块提供入口 |
| P2 | 客户联系人与组织关系、附件与文档管理 | 补齐客户经营和业务资料沉淀基础 |
| P3 | 报价/价格/方案管理 | 补齐商机到合同之间的关键销售动作 |
| P4 | 数据导入导出、日报/周报管理 | 提升运营效率和管理复盘能力 |

## 非本阶段范围

以下能力仍重要，但不纳入本阶段主目标：

- 完整财务闭环
- 合同审批流
- 通用流程引擎
- 销售预测 Forecast
- 库存/资产管理
- 金蝶集成正式启用

这些模块后续应作为独立阶段规划。
